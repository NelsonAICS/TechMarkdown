# 启动页与批注系统重构设计

日期：2026-07-16

## 目标

本次重构解决三类问题：

1. 启动页信息层级混乱，底部操作区挤压最近文件，图标和文字在多层合成后发虚。
2. 启动页持续运行粒子、模糊渐变和玻璃材质，窗口空闲时仍以固定频率重绘。
3. 批注入口、编辑、定位和列表分散在编辑器、预览、系统弹窗与 AI 侧栏中，操作不连续，且每次输入会重复扫描全文。

## 方案取舍

评估过三种方向：

- 保留原布局，仅调整颜色和间距：改动小，但无法解决持续合成和信息架构问题。
- 保留动态背景，降低粒子帧率：只能缓解 GPU 压力，文字仍可能被离屏栅格化。
- 改为静态原生工作台，并把批注升级为独立审阅工作区：结构更清楚，也能从根源降低合成与刷新开销。

最终采用第三种方案。

## 启动页

启动页分为两个稳定区域：

- 左侧“开始”区：品牌、新建 Markdown、打开文件、打开文件夹、LaTeX/HTML 快捷入口。
- 右侧“继续工作”区：最近项目和最近文档，统一放在可滚动内容区。

渲染约束：

- 不使用 `TimelineView`、持续动画、`blur`、`material` 或 `drawingGroup`。
- 背景只保留静态渐变和低透明度网格。
- 文字与 SF Symbols 不参与缩放动画，避免栅格化发虚。
- 文件卡片使用实色层、单层边框和轻量悬停反馈。
- 默认窗口按 980×720 离屏渲染验收。

## 批注架构

### 数据与定位

`AnnotationService` 是唯一写入和变更通知来源。它在锁内修改数据，再把不可变快照交给串行持久化队列，避免后台编码与前台写入同一字典。

`AnnotationLocator` 统一编辑器跳转与高亮定位，按以下顺序查找：

1. 创建时的 UTF-16 行列快照与原文完全一致；
2. 文本移动后，在原位置附近寻找最近匹配；
3. 使用上下文消除重复文本歧义；
4. 退化为直接文本匹配；
5. 无法定位时返回失效状态，不错误跳转到其他同名文本。

已解决批注不会阻止在同一位置创建新批注；文档级意见没有选区，因此不会被误判为重叠。

### 交互

AI 侧栏分为“对话”和“批注”两个工作区。

选中文字后执行“添加批注”：

1. 自动展开 AI 侧栏；
2. 自动切换到批注工作区；
3. 在侧栏内显示选中原文和批注编辑器；
4. 保存后立即进入“待处理”列表；
5. 点击原文引用可同时定位编辑器与预览。

批注状态明确显示为：

- 已定位
- 已重定位
- 原文已变化
- 全文意见
- 已解决

预览中的批注高亮可点击并打开批注工作区。删除操作需要二次确认；编辑、解决、重新打开和删除集中在同一菜单中。

## 性能策略

- 编辑器输入时不再同步执行两次全文批注扫描；批注高亮延迟 180 ms 合并刷新。
- 预览渲染延迟 120 ms 合并连续输入，主题和缩放仍即时更新。
- 编辑器高亮只使用统一定位器返回的目标范围，不再对每条批注重复高亮所有同名文本。
- 批注高亮不覆盖正文前景色，减少文字颜色冲突。
- 最近文件索引改为串行、确定性读改写，避免启动页读取到尚未落盘或互相覆盖的数据。

- `swift build`：通过。
- `swift test`：44 项逻辑测试通过；另有 2 项按环境变量启用的离屏渲染测试。
- 批注定位覆盖：精确位置、文本移动、重复文本上下文消歧、原文删除。
- 启动页按 980×720 离屏渲染检查。
- 批注侧栏按 400×720 离屏渲染检查。
